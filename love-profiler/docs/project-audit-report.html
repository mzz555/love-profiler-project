CRITICAL 阻塞上线 · 6 项
C-01 广告 token 验签逻辑错误 — 自指比较恒为 False CRITICAL
app/api/unlock.py:46-47
影响 hmac.new(secret, ad_token).hexdigest() 的结果与 ad_token 本身比较 — 永远不等。加上 L42 缺密钥时 return True，任何人调 /unlock/ad 都直接通过，广告解锁形同虚设。
修改 客户端传 ad_token + signature 两个字段；服务端用 hmac(secret, ad_token) 算出 expected 比对 signature。或接抖音官方服务端到服务端激励视频回调，废弃客户端 token。
C-02 验签 fail-open — 缺密钥时默认放行 CRITICAL
unlock.py:42-44 · pay.py:117
影响 DOUYIN_AD_SECRET 或 DOUYIN_PAY_TOKEN 未配置时，验签逻辑直接跳过。生产忘配 = 广告解锁免验 + 支付回调可伪造。
修改 改为 fail-closed：缺密钥时在 main.py lifespan 启动期 raise RuntimeError 或至少 return False（拒绝请求），绝不能 return True。
C-03 前端写死 ad_token='ad-complete' CRITICAL
miniprogram/pages/unlock/unlock.js:39
影响 前端把硬编码字符串 'ad-complete' 当 ad_token 传给后端。配合 C-01/C-02 = 完整的白嫖攻击链：用户根本不看广告就能解锁报告。
修改 接抖音激励视频 SDK 的 onClose 回调中获取真实 verify token；或改用服务端到服务端回调（抖音 server 通知你的后端，前端只传 transaction_id）。
C-04 CORS allow_origins=["*"] CRITICAL
app/main.py:64-69
影响 所有源均可跨域访问全部 API，包括 /admin 管理面板。攻击者可在任意网页中用 JS 发请求窃取 admin 数据。
修改 生产环境 allow_origins 改为显式白名单（你的域名）。小程序请求不依赖 CORS，直接移除或限定到 admin 域名。
C-05 BASE_URL 硬编码局域网 HTTP IP CRITICAL
miniprogram/app.js:9
影响 http://192.168.3.179:8000 写死在代码里。上线后所有用户连不上后端。且 HTTP 明文传输 JWT token。
修改 引入构建期环境变量注入（开发者工具支持 project.config.json 条件编译），或至少抽到 config.js 并在部署清单中加 CI 检查拦截。
C-06 DATABASE_URL 默认 sqlite，生产不报错 CRITICAL
app/database.py:10
影响 DATABASE_URL 未配置时默认用 sqlite，不抛异常。生产多 worker 并发下 sqlite 会锁死，数据丢失。
修改 在 lifespan 启动期检查：如果 DATABASE_URL 以 sqlite 开头且 DEV_MODE != true，直接 raise RuntimeError("生产环境禁止使用 sqlite")。
HIGH 部署前必修 · 9 项
H-01DEV_MODE 误开 = 全栈失守HIGH
dev_auth.py · dev_pay.py · access_control.py · admin.py
影响一个环境变量 DEV_MODE=true 同时开启：免登录签 JWT、模拟支付、免付费墙、admin 免鉴权。生产偶然配错即全线暴露。
修改在 lifespan 中加 hostname/IP 白名单校验；DEV 路由绑 127.0.0.1 only；加启动日志醒目大字警告 ⚠ DEV_MODE IS ON。
H-026 条路由无限流保护HIGH
pay.py · unlock.py · admin.py · dev_auth.py · ws_result.py
影响/pay/* /unlock/ad /admin/* /ws/result 均无 @limiter.limit。攻击者可无限刷订单、刷 LLM、批量解锁。
修改每条路由补 @limiter.limit：pay 5/min、unlock 5/min、admin 30/min、ws 3/min。WS 额外加并发连接数限制。

修改引入 pydantic BaseSettings 集中管理所有环境变量，启动时一次性校验、类型转换、缺省值兜底。
H-06history 用正则从 LLM 文本反解 type_nameHIGH

H-07randomPick 一键随机答题污染数据HIGH
miniprogram/pages/chat/chat.js:172-185

修改加 if (!app.isDev) return; 守卫，或直接从生产代码中删除该函数和对应 UI 按钮。
H-08pay.create_order payload app_id 重复HIGH

H-09adUnitId 仍为占位字符串HIGH

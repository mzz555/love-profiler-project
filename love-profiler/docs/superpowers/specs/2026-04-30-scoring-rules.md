# 评分规则说明文档（v0）

**来源：** 规则说明文档 PDF（2026-04-30）  
**适用范围：** Agent A 诊断逻辑、answer_package_builder 组包规则

---

## 第一部分：5 个维度的内在结构

```
关系底色层（最深层）
  D1 依恋行为 —— 用户的依恋类型（Bowlby & Ainsworth 依恋理论）

关系运作层（中间层）
  D2 边界意识 —— 与人的距离感（Bowen 自我分化 + Cloud & Townsend 边界理论）
  D3 冲突处理 —— 摩擦时的反应（Gottman 四骑士 + EFT 情绪聚焦法）
  D4 情感需求 —— 怎样被爱才有感（Chapman 五种爱的语言）

表层呈现（最表层）
  D5 风格表达 —— 怎么说话怎么分享（沟通风格量表 + 抖音/小红书内容观察）
```

**为什么需要跨维度审查：** D5 看起来"高分享、高直接"很健康，但 D1 是焦虑型——那他的"高分享"很可能不是真分享，而是焦虑驱动的「填空式存在确认」。单维度看不出，组合起来才看得出。

## 第二部分：每个维度测什么

### D1 依恋行为（6 道题）

**测什么：** 遭遇"不确定性"时依恋系统的激活模式（对方延迟回复/突然冷淡/出现第三方时的内心反应）

**子信号：**

| 信号 | 内核 | 题号 |
|------|------|------|
| S1 不确定性解读 | 对方"正在输入..."又消失，第一念头 | D1-Q01 |
| S2 距离容忍度 | 周末没联系，能在多久内不焦虑 | D1-Q02 |
| S3 第三者敏感度 | 看到对方朋友圈出现新异性的反应 | D1-Q03 |
| S4 激活后表达（核心） | 对方冷淡时，直接沟通还是回避 | D1-Q04 |
| S4 激活后表达（验证） | 同上信号的另一情境，做交叉验证 | D1-Q05 |
| S5 安全感来源 | 什么样的相处模式给你"被坚定选择"的感受 | D1-Q06 |

**输出：** 5 类标签（安全 / 中度安全 / 混合 / 中度焦虑 / 焦虑）

**S4 出两题的原因：** 用户都知道"应该直接沟通"，两题互证识别出"嘴上说会沟通、行为上却回避"的用户（社会期望污染防御）。

---

### D2 边界意识（6 道题）

**测什么：** 用户能否在关系中保持"独立的自我"，能否对越界行为做出健康响应

**子信号：**

| 信号 | 内核 | 题号 |
|------|------|------|
| S1 越界识别（核心） | 对方翻你手机后坦白，你的反应 | D2-Q01 |
| S2 自我维持 | 在恋爱中，事业/朋友圈/独处时间是否被吞噬 | D2-Q02 |
| S3 异性相处共识 | 对伴侣的异性社交，你们怎么处理 | D2-Q03 |
| S4 付出对等 | 经济上、情绪上、付出上是否平衡 | D2-Q04 |
| S1 越界识别（验证） | 持续性贬低/翻旧账时的反应 | D2-Q05 |
| S5 隐私空间 | "情侣应该互相透明"还是"应该有自己空间" | D2-Q06 |

**输出：** 3 类标签（清晰 / 混合 / 模糊）

**S1 出两题的原因：** Q01 测"对单次越界能否响应"，Q05 测"对持续性伤害能否响应"。常见诊断模式：Q01 选 A/B（健康），Q05 选 C/D（失语）→「单次能识别、持续中失语」，是边界维度最重要的复合诊断信号之一。

---

### D3 冲突处理（6 道题）

**测什么：** 关系中出现摩擦时的反应模式（关系长期健康度的最强预测变量，基于 Gottman 研究）

**子信号：**

| 信号 | 内核 | 题号 |
|------|------|------|
| S1 表达方式（核心） | 小事情绪不爽时，你怎么开口 | D3-Q01 |
| S2 修复主动性 | 冷战时，谁先递台阶 | D3-Q02 |
| S3 责任归因 | 吵完冷静后，你认为问题在谁 | D3-Q03 |
| S4 情绪淹没管理 | 情绪即将过载时，你怎么处理 | D3-Q04 |
| S1 表达方式（验证） | 重大分歧（关于未来）下的表达 | D3-Q05 |
| S5 追逃模式中的角色 | 历史摩擦中，你扮演追还是逃 | D3-Q06 |

**输出：** 5 类标签（健康 / 中度健康 / 混合 / 中度问题 / 问题）+ 追逃亚型（追/逃/觉察打破/无固定模式）

**D3-Q06 特殊规则（⚠️ 关键）：**
- A（觉察打破循环）= +2，score_meta = {}
- B（无固定模式）= +1，score_meta = {}
- C（追的角色）= -2，**score_meta = {"pursue_avoid": "pursue"}**
- D（逃的角色）= -2，**score_meta = {"pursue_avoid": "avoid"}**

C 和 D 同样是 -2，但追型和逃型用户即使冲突维度总分一样，报告需要走完全不同的解读路径。

---

### D4 情感需求（6 道题）

**测什么：** 用户最能感知到"被爱"的方式——五种爱的语言中的偏好排序

**题型设计：**

| 题号 | 题型 | 测什么 |
|------|------|--------|
| Q01 | 主导偏好题 | 用户主动选择的"最强爱的语言"（5选1，A-E对应T1-T5，每选+2） |
| Q02 | 对比题 T1 vs T4 | 言语肯定 vs 服务行动 |
| Q03 | 对比题 T2 vs T5 | 精心时刻 vs 身体接触 |
| Q04 | 对比题 T3 vs T4 | 用心小惊喜 vs 服务行动 |
| Q05 | 对比题 T1 vs T2 | 言语肯定 vs 精心时刻 |
| Q06 | 验证补盲题 T3 vs T5 | 用心小惊喜 vs 身体接触 |

**对比题打分示例（D4-Q02，T1 vs T4）：**
- A → T1+2
- B → T4+2
- C → T5+1（次选）
- D → T2+1（次选）

次选选项也给 +1 分，用于识别"主选 T1，但 T2 也是次要需求"的精细偏好。

**输出：** 5 类爱的语言的相对偏好排序（top2 组合，例如"T1+T4"）

**⚠️ D4 归一化（非常关键，不要跳过）：**

由于题库设计的不对称性，T3 最大可得分只有 6 分，其他类型可达 8-9 分：

| 类型 | 最大可能得分 |
|------|------------|
| T1 言语肯定 | 9 |
| T2 精心时刻 | 8 |
| T3 用心小惊喜 | **6（特别低）** |
| T4 服务行动 | 9 |
| T5 身体接触 | 8 |

**归一化公式：** 每类得分 ÷ 该类最大可能得分 = 0~1 之间的相对偏好度

直接看原始得分会系统性低估 T3 偏好用户。Agent A 必须做归一化后再取 top2。

**自我认知一致性检查：**
- D4-Q01（用户主动选的"最强需求"）vs 归一化后 top1
- 一致 → `love_language_self_awareness = "aligned"`
- 不一致 → `love_language_self_awareness = "misaligned"`（Agent B 触发"自我认知盲区"段落）

### D5 风格表达（6 道题）

**测什么：** 两个独立子面——直接性（S1）+ 分享欲（S2）

**S1 直接性（3 题）：**

| 题号 | 测什么 |
|------|--------|
| Q01 | 日常需求（想吃什么/想要什么）的直接表达 |
| Q02 | 日常情绪（不舒服时）的直接表达 |
| Q03 | 关系定位（暧昧期捅破窗户纸）的直接表达 |

**S2 分享欲（3 题）：**

| 题号 | 测什么 |
|------|--------|
| Q04 | 对有趣事物的分享欲 |
| Q05 | 下班后日常状态的分享欲 |
| Q06 | 被追问时的回应方式 |

**输出：** 2x2 象限（直接×分享）

|  | 高分享 | 低分享 |
|--|--------|--------|
| **高直接** | 直爽热情型 | 清爽利落型 |
| **高含蓄** | 碎碎念含蓄型 | 安静含蓄型 |

中间 5 个过渡象限以最近邻名称 + "偏中"标记，例如"直爽热情型偏中"。

**风险监控：** 如果内测后 S1 和 S2 相关系数 > 0.8，说明两子面不独立，需回退到单维度方案。

## 第三部分：打分和计算逻辑

### 3.1 三种打分模型

| 维度 | 打分模型 | 输出 |
|------|----------|------|
| D1 依恋、D2 边界、D3 冲突 | 强度型（intensity） | -12 到 +12 的连续分数 → 映射成 3-5 类标签 |
| D4 情感需求 | 类型偏好型（typology） | 五类相对偏好排序 → top2 组合 |
| D5 风格表达 | 双子面型（dual facet） | S1 分数 × S2 分数 → 2x2 象限定位 |

---

### 3.2 强度型打分（D1 / D2 / D3）

**Step 1：题目级打分**

| 选项 | 分数 | 含义 |
|------|------|------|
| A | +2 | 健康端（安全/清晰/健康） |
| B | +1 | 中度健康 |
| C | -1 | 中度问题端 |
| D | -2 | 问题端 |

**Step 2：维度级累加**  
6 道题分数全加 → raw_total，范围 -12 到 +12

**Step 3：标签映射**

```
raw_total ≥ 6          → 健康端    （依恋安全型 / 边界清晰 / 冲突健康）
3 ≤ raw_total < 6      → 中度健康  （依恋中度安全 / 冲突中度健康）
-3 ≤ raw_total < 3     → 混合
-6 ≤ raw_total < -3    → 中度问题端（依恋中度焦虑 / 边界模糊）
raw_total < -6         → 问题端    （依恋焦虑型 / 冲突问题）
```

**D5 子面标签映射（范围 -6 到 +6，每子面 3 题）：**

| 总分范围 | S1 标签 | S2 标签 |
|----------|---------|---------|
| > 3 | 高直接 | 高分享 |
| -3 ~ 3 | 中直接 | 中分享 |
| < -3 | 高含蓄 | 低分享 |

---

### 3.3 类型偏好型打分（D4）

详见第二部分 D4 节。核心步骤：
1. 按选项指向的爱的语言类型打分（非 A/B/C/D 位置）
2. 累加 T1-T5 各自的原始分
3. 归一化（÷ 最大可能得分）
4. 取 top2
5. 自我认知一致性检查（D4-Q01 主动选 vs 归一化 top1）

---

### 3.4 跨维度审查（三层）

#### 层级 1：维度内交叉验证（3 条规则）

| 规则 | 触发条件 | 含义 |
|------|----------|------|
| CV-D1-S4-一致性 | Q04 vs Q05 分数差距大 | 用户社会期望偏差，S4 信号不可信 |
| CV-D2-S1-觉察缺口 | Q01 选 A/B + Q05 选 C/D | 对单次越界能识别，对持续伤害失语 |
| CV-D3-S1-压力韧性 | Q01 ≥ +1 + Q05 < 0 | 小事能软启动，重大分歧下表达能力崩溃 |

#### 层级 2：维度间交叉验证（3 条规则）

| 规则 | 触发条件 | 复合诊断 |
|------|----------|----------|
| CV-跨-D2D3-S1 | D2-Q01 = -2 + D3-Q01 = -2 | 外强内弱：对越界无响应，对小事却攻击性表达 |
| CV-跨-D1D5-S2 | D1-Q02 = -2 + D5-Q05 = -2 | 焦虑型回避伪装：对方不联系焦虑，自己却不主动 |
| CV-跨-D2D5-S1 | D2-Q02 ≤ -1 + D5-S1 多题选 C/D | 自我溶解风险：在关系里既保不住自己也不会表达 |

#### 层级 3：全局复合诊断（4 个标记）

| 标记 | 触发条件 | 含义 |
|------|----------|------|
| `awareness_gap_global` | D1-Q01-D + D2-Q05-D + D3-Q03-D 中至少 2 个 | 跨三维度"自我归因式-2"模式：依恋自我威胁化 + 边界被伤害自我攻击 + 冲突责任全揽 |
| `pursue_avoid_role` | D3-Q06 选项 | 追/逃/觉察打破/无固定模式四类 |
| `stable_personality` | 30 题中 +1/+2 占比 ≥ 60% | 稳定型用户（触发"优势版"报告） |
| `love_language_self_awareness` | D4 主动选 vs top1 是否一致 | 是否存在自我认知盲区 |

**为什么层级 3 最关键：** `awareness_gap_global = true` 把"焦虑型"变成"焦虑型 + 跨维度自我归因模式"——这是有诊断深度的复合画像。Agent A 严格按规则触发这些复合标记，Agent B 把标记翻译成有温度的洞察。中间不能被"语感"软化，否则诊断锐度全部丧失。

### 3.5 16 类核心分型

**组合公式：**  
依恋（4 类：S 安全 / MS 中度安全 / MA 中度焦虑 / A 焦虑）  
× 边界（2 类：CL 清晰 / BL 模糊）  
× 冲突（2 类：H 健康 / P 问题）  
= **4 × 2 × 2 = 16 类**

**type_code 示例：** `"MA-CL-MH"`（中度焦虑依恋 / 清晰边界 / 中度健康冲突）

**D4 / D5 不进入核心分型：** 加进去会变成 1440 类。产品策略：
- 16 类 = 主类型名（社交分享用）
- D4 + D5 = 子画像（报告内深度个性化，让同一类型用户有差异感）

**依恋维度的 4 类粗分映射（标签缩写对应关系）：**

| raw_total 范围 | 标签 | 缩写 |
|--------------|------|------|
| ≥ 6 | 安全型 | S |
| 3 ~ 5 | 中度安全 | MS |
| -2 ~ 2（含-3~2） | 混合型 | M（注意：设计文档用 M，16类分型合并后统一） |
| -3 ~ -5 | 中度焦虑 | MA |
| ≤ -6 | 焦虑型 | A |

> 注：16 类分型中依恋取 4 档（S/MS/MA/A），"混合型 M"在 v0 中归入 MS 或 MA 由 Agent A 按得分就近归类。

## 第四部分：完整算分例子（小 L）

小 L 的答题（30 道）走通全流程：

**D1 依恋：** -1+(-2)+(-1)+2+(-1)+1 = **-2** → 混合型（M）  
**D2 边界：** 2+1+2+2+(-1)+1 = **+7** → 边界清晰（CL）  
**D3 冲突：** 2+2+1+2+(-1)+(-2) = **+4** → 中度健康（MH）+ 追逃亚型：追  
**D4 情感：** T1=7/9=0.78, T2=2/8=0.25, T3=0, T4=2/9=0.22, T5=0 → top2=T1+T2（言语肯定+精心时刻），aligned  
**D5 风格：** S1=+3（中直接）, S2=+3（中分享）→ 直爽热情型偏中

**跨维度审查结果：**
- CV-D1-S4-一致性：Q04(+2) vs Q05(-1)，差距=3 → LOW（社会期望偏差）
- CV-D2-S1-觉察缺口：Q01=A + Q05=C → TRUE（单次觉察，持续中失语）
- CV-D3-S1-压力韧性：Q01(+2) + Q05(-1) → LOW（压力下表达崩溃）
- awareness_gap_global：FALSE（D选项数 <2）
- pursue_avoid_role："pursue"（追型）

**最终分型：** M-CL-MH → Agent B 类型名翻译

## 第五部分：内测风险监控指标

| 层面 | 风险点 | 监控阈值 |
|------|--------|----------|
| 题库 | D2-Q03 选项 B "默契上彼此知道分寸"区分度不足 | 内测后统计 B 选率，>60% 需调整 |
| 题库 | D3-Q06 选项 B "无固定模式"可能是无觉察伪装 | 用 D3-Q02/Q04 得分交叉验证 |

| 算法 | awareness_gap_global 触发率 | 合理范围 10-20% |
| 算法 | 16 类分型分布 | 某一类不应超过 30% |
| 算法 | D4 归一化效果 | T3 偏好占比应与 T1/T2/T4 量级相当 |
| 报告 | 诊断锐度被语感软化 | 人工抽检 10% 报告 |
